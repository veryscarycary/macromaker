  <action>Human visual verification required — cannot be automated. See how-to-verify steps below.</action>
  <how-to-verify>
    **iOS Simulator (required):**
    The app should be running from Task 1. Look at the smoke test screen:
    1. Are the 4 weight sections visually distinct? Regular should look thinner than Bold.
    2. Is the text clearly Inter (geometric sans-serif) — NOT the system San Francisco font?
    3. Does the "Display" size text in the type scale look substantially larger than "Caption"?
    4. Does the orange color swatch appear (brand.primary = #f97316)?
    If any text section looks identical in weight to another, fonts are NOT loading correctly.

    **Android Simulator (required):**
    Run `npx react-native run-android` in a terminal and repeat the same visual checks.
    Android failure mode: All 4 sections render at the same weight (fallback to Roboto Regular).

    **Physical iOS Device (required — per STATE.md blocker):**
    Connect your iOS device, open Xcode, select your device, and run the app.
    Repeat the same visual checks. Simulator and device can differ in font resolution behavior.

    **What to look for:**
    - PASS: 4 clearly different stroke weights; geometric/grotesque letterforms
    - FAIL: All weights look the same (font not loading — check pod install, clean build)
    - FAIL: Text shows boxes or "?" characters (font file corrupt or misnamed)
  </how-to-verify>
